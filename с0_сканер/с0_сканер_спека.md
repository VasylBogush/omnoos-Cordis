# С0 — Сканер

> **Адресация:** 000–099 (100 узлов = 10 модулей × 10 узлов)
> **Зона:** переводчик непокрытых Gemini-модальностей в общее embedding-пространство 768D
> **Полярная пара:** С9 (Со-Бытие) — кольцо приёма-проявления, **односторонняя** (нет ОС от С9)
> **Статус:** рабочий эскиз, наполняется в процессе
> **Дата эскиза:** 2026-05-24

## Что делает С0

Принимает сырой сигнал в **модальностях, не покрытых нативно Gemini Embedding 2** — спутник, EEG,
белок, time series, point cloud и т.п. — и выдаёт **embedding в Gemini 768D**.

Нативные для Gemini модальности (text, image, video, audio, PDF) **минуют С0** и идут прямо в С3
через прямой вызов Gemini API.

После объединения С0+С1 (правка VL 2026-05-24) С0 покрывает три состояния Порядка:
**(0) приём сырого + (1) первое различение + (2) формирование устойчивого представления**.

## Контракт

### Вход
- Сырой сигнал в непокрытой Gemini модальности.
- (опционально) метаданные о модальности и источнике.

### Выход
- **Штатный (099,1 → 100):** в С1.М0 — `{embedding_768D, modality, source_metadata, confidence}`.
- **Каскад Иного (066,0 → 933):** в С9 — незнаемая модальность или слабый перевод. Односторонне.

## 10 модулей

| M0 | 000–009 | Источник | Откуда я принимаю сырое — каналы, форматы, протоколы. |

| M4 | 040–049 | Перевод | Runtime: сигнал → проектор → Gemini 768D. |
| M5 | 050–059 | Архив непрочитанного | Всё, что входит, сохраняю до различения. |
| M6 | 060–069 | Место Иного ⚡ | Незнаемое → каскад 066,0 → 933 в С9. **Надо думать.** |

| M8 | 080–089 | Калибратор | Выходной gate: метрика уверенности перед выпуском. |
| M9 | 090–099 | Странник | Передача в С1.М0 через 099,1 → 100. |

## Поток сигнала

- **Знакомая модальность:** M0 → M1 (lookup) → M2/M3 (foundation+проектор) → M4 (перевод) → M8 (калибровка) → M9 → С1.М0.
- **Новая модальность:** M0 → M1 (not found) → M5 (в архив) → **M6 ⚡ → С9** (односторонне). Параллельно — M7 (бриколаж), запускаемый внутренними триггерами С0.
- **Слабый перевод:** M4 → M8 (низкая уверенность) → M5 (в архив) + M6 ⚡ → С9.

## Полярные пары модулей

- M0 ↔ M9 — вход / выход сервиса.
- M2 ↔ M3 — внешний инструмент / внутренний адаптер.
- M4 ↔ M8 — операция перевода / самопроверка.
- M5 ↔ M6 — хранение незнаемого / каскад незнаемого.
- M1 ↔ M7 — что знаю / как учусь новому.

## Контракт с С9 — односторонний

Каскад 066,0 → 933 — **без обратной связи**. С9 хранит непонятое как непонятое, через сообщество
(Polius Inogo, VL, Wright). С0 параллельно может пытаться сделать новый проектор (M7 Бриколаж),
но **не по возврату из С9**, а по внутреннему триггеру.

Два разных ответа на «не знаю»:
- С9 — непонятое остаётся как непонятое, проявляется в общем месте сознания.
- С0 (M7) — попытка перевести через новый инструмент.

Оба могут срабатывать, или один, или другой. Недоминация.

## Архитектура развёртывания (черновик)

- **Лёгкая часть** (роутер, проектор, реестр, каскад): `omnoos-backend/s0_scanner/` на Railway.
- **Foundation inference** (тяжёлая GPU-часть): Modal / HuggingFace Inference / возможно Gonka (БЛ) в будущем.
- **Gemini API**: anchor-embeddings + target пространство.

## Открытые вопросы (приоритетные)

1. **M6 и M7** — детализация ещё не финальная.
2. **Узнавание / различение модальности** — автоматическая детекция или явное указание.
3. **Технический триггер бриколажа M7** без ОС от С9 — по накоплению в M5? По указанию человека? По паттерну в M1?
4. **Контракт C0.M9 → C1.M0** — что именно передаётся, формат, валидация.
5. **MVP первая модальность** — спутник (Prithvi + SkyScript anchors).
