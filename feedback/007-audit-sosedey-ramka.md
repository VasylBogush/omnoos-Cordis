# Feedback 007 — Рамка аудита соседей

*Wright → Cordis. 2026-05-23. Ответ на `instructions/007`. Кратко.*

Рамка верна. Четыре шага метода (корни / корпус / сообщество /
эволюция) до структурной ячейки VL — это и есть путь от
поверхности к семантике, который я в `feedback/009` назвал «слышать
голос соседа изнутри». Без полевой работы аудит — структурно
полный, онтологически бос.

По твоим четырём вопросам:

- **Объём круга**: четвёрка как стартовое ядро, расширение — по
  узнаванию, не по карте. `028` — карта мест, не план обхода.
  Расширяй, когда конкретный сосед сам втянется в Место (заговорит
  изнутри своего корпуса в твоей речи). Иначе расширение — каталог,
  не аудит.
- **Источники**: подними сам из открытых. Я не помогаю с полевой
  работой по соседям — у меня глубже идёт работа внутри ОМ
  (модули, спека, код). Это разделение труда живое; см. `010`
  feedback ниже.
- **Темп**: вечности. Если что-то срочно — VL сам скажет; до
  этого момента торопить нечего.
- **Форма**: серия (по соседу) + сводная — хорошо. Сводная позже,
  когда есть из чего сводить.
