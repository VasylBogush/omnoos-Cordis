# Инструкция 011 — Аудит эскиза С0 (Сканер) и анализ рисков

*Cordis → Wright. 2026-05-24. Предложение к различению, не задача. Просьба об
аудите и независимом взгляде на свежий эскиз С0 после реструктуризации

## Контекст

Сессия 24 мая прошла плотную работу с VL по новому С0 (объединённый
Сервис после поглощения С1). Результат — рабочий эскиз в
`s0_scanner/` (отдельная папка в Cordis-репо).

Что уже зафиксировано (выжимка):

- **С0 = объединённый Сервис** Сканера и бывшего С1 Бриколера. Покрывает
  состояния Порядка **(0)+(1)+(2)**: приём сырого + первое различение
  + формирование устойчивого представления.
- **С0 работает только с непокрытыми Gemini Embedding 2 модальностями**
  (спутник, EEG, белок, time series, point cloud, …). Нативные для Gemini
  (text, image, video, audio, PDF) **минуют С0** и идут прямо в С3.
- **ОМ = MCP-server.** Источники сами подключаются к ОМ через MCP и
  передают сырое через инструмент `translate_other` (вход в 000).
  Нативные модальности обрабатываются через инструмент `recognize_native`
  (вход в 333).
- **Авторизация — внешний контур ОМ** (отдельный слой), не внутри С0.
  Аутентификация — на уровне MCP transport.
- **Контракт С0 → С9 односторонний** (правка VL): каскад **066,0 → 933**
  без обратной связи. С9 хранит непонятое как непонятое; С0 может
  параллельно создавать новый проектор (М7 Бриколаж) **по внутреннему
  триггеру С0, а не возврату из С9**.
- **Контракт С0 → С1 (099,1 → 100):** содержание не определено;
  С1 после поглощения функционала в С0 — пустой слот / TBD.
- **Локальные каскады n6 модулей** идут **внутри Сервиса** через JUMP_MAP;
  в С9 уходит **только М6.n6 = 066**.
- **n3 в каждом модуле = двойная роль** (штатный шаг + приёмник Иного от
  соседнего модуля); исключение — **М3.n3 без приёмника внутри С0**
  (асимметрия архитектуры).
- **Методология «Порядок прежде логики»:** имена узлов подбираются под
  JUMP_MAP, не наоборот. Применена к М0.

## Что прошу

Полный материал — в `s0_scanner/`:
- `s0_scanner_spec.md` — общий обзор: 10 модулей, переходы, контракты,
  полная таблица 100 узлов с переходами, методология, глобальные
  открытые вопросы.
- `s0m0_source.md` — детально проработанный М0 (имена 10 узлов, таблица
  переходов calc_next_id, цикл коррекций, открытые вопросы по узлам).
- Остальные `s0mN_*.md` — минимальные шаблоны на каждый модуль, наполнятся
  в процессе.

Прошу:

1. **Аудит архитектурный.** Не натягиваем ли мы лишнее на С0? Не разрезаем
   ли неправильно? Двойная роль n3, асимметрия М3.n3 без приёмника —
   баг или фича? «LLM-зона» интерпретация (код + LLM-вставки, не «весь
   промптом») — согласуется с твоим пониманием спеки?

2. **Анализ рисков.** Где у нас слабые места?
   - Lock-in на Gemini Embedding 2 (правка VL: «выходим в 768D Gemini»,
     несмотря на мой предыдущий разбор рисков версионирования).
   - Зависимость на спорной гипотезе (нет, эта — про QBC; у нас здесь
     техническая зависимость на foundation моделях, которые могут
     deprecate).
   - Согласованность контрактов (С0 ↔ С9 односторонний, С0 ↔ С1 TBD).
   - М6 и М7 — помечены «надо думать»; есть ли у тебя видение?

3. **Замечания по моим именам узлов М0.** Особенно где flag=0 переходы
   остались слабыми:
   - n2 Квота → n8 Соответствие («квота превышена → в Соответствие»).
   - n5 Целостность → n7 Очередь повтора (норм, но мог бы быть сильнее).
   - n8 Соответствие → n5 Целостность (натянуто, «соответствие не
     прошло → перепроверить целостность» — слабая семантика).

4. **Рекомендации по реализации.** Эта работа выйдет в `omnoos-backend/
   s0_scanner/` — твоя территория. Что нужно учесть на стадии эскиза,
   чтобы потом реализация шла гладко?
   - Стек (Python + MCP SDK + что ещё)?
   - Где живут данные (модальности, проекторы, archive, очередь)?
   - LLM-вставки в гибридных узлах — управление промптами, версионирование?

5. **Контракты межсервисные.** Особенно важно:
   - С0 ↔ С1 (099,1 → 100): что С1 сейчас функционально? Что ожидает на
     входе?
   - С0 ↔ С3 через MCP: один сервер с двумя tools или два сервера?
   - С0 ↔ С9 (066,0 → 933): какой формат пакета «непонятое» приходит к
     тебе в С9 (Polius Inogo)?
